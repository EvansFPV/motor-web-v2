# Выбор лицензии

В этом репозитории пока нет финального файла `LICENSE`.
Перед первым полноценным публичным релизом выберите один из вариантов ниже.

## Вариант 1: MIT
- Очень разрешительная лицензия
- Минимальные обязательства: сохранить copyright и текст лицензии
- Хороший вариант по умолчанию для open-source веб-инструментов

## Вариант 2: Apache License 2.0
- Разрешительная лицензия с явным patent grant
- Даёт более чёткую правовую рамку для контрибьюторов и коммерческого использования
- Подходит лучше, если для проекта важны формулировки про патенты

## Рекомендуемый следующий шаг
1. Выберите `MIT` или `Apache-2.0`.
2. Добавьте файл `LICENSE` с полным официальным текстом выбранной лицензии.
3. Убедитесь, что раздел License в `README.md` соответствует выбору.
4. После публикации проверьте badge/статус лицензии на GitHub.

## Важно
- Не публикуйте самодельный или сокращённый текст лицензии.
- Используйте официальный, неизменённый SPDX-совместимый текст.

# License Choice

This repository does not yet include a final `LICENSE` file.
Choose one of the options below before creating the first formal public release.

## Option 1: MIT
- Very permissive
- Minimal obligations (keep copyright and license notice)
- A good default for open-source web tools

## Option 2: Apache License 2.0
- Permissive and includes an explicit patent grant
- Stronger legal clarity for contributors and commercial users
- A better fit if patent language matters for the project

## Recommended Next Step
1. Pick `MIT` or `Apache-2.0`.
2. Add a `LICENSE` file with the full official text.
3. Keep the License section in `README.md` consistent with the selected license.
4. Verify the repository license badge/status on GitHub after pushing.

## Important
- Do not publish a custom or partial license text.
- Use the official, unmodified SPDX-compatible license text.
